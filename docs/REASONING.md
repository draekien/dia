# Reasoning Log

This is an index of non-obvious decisions, gotchas, and learnings recorded during development sessions. Unlike `docs/adr/`, which captures significant *architectural* decisions, `docs/reasoning/` captures smaller-grained findings — a tricky bug's root cause, a library quirk, a rejected approach and why — that don't warrant a full ADR but are worth remembering so they aren't rediscovered the hard way.

- `template.md` — the template used for new entries.
- `YYYY-MM-DD-short-slug.md` — one entry per learning, dated and named for what it's about.

| 2026-07-18 | [Wiring `useChat` into the pane](reasoning/2026-07-18-usechat-pane-wiring.md) *(superseded by ADR-0015)* | `useChat` `connect()` is lazy (never fires on mount unless `live` — don't set it, our `connect` sends the user message); `initialMessages` is read once at client creation, so gate on the async history query and mount `PaneChat` with `key={paneId}`; message timeline lives in `useChat` while attention/permission/question stay in TanStack Query. Because `useChat` state is component-local, a split remounts the pane and blanks it — mirror `chat.messages` into the Query cache and reseed `initialMessages` from that snapshot (`resolveInitialMessages`). | 2026-07-19 | [`setTitleBarOverlay` rejects CSS Color 4 strings](reasoning/2026-07-19-titlebar-overlay-color4-parse.md) | Chromium serializes `getComputedStyle().color` of an `oklch` token as a Color 4 string (`oklab()`/`oklch()`); Electron's native overlay color parser only groks legacy formats and throws an uncaught main-process `Could not parse color as CSS color`. | 2026-07-19 | [Syncing `/clear` and `/compact` from the pane session](reasoning/2026-07-19-slash-command-clear-compact-sync.md) | Command list comes from two SDK messages (`init` names-only, `commands_changed` enriches — replace, don't merge); `/clear`'s `conversation_reset` must persist `new_conversation_id` via `onSessionId` or resume reloads the old transcript, and it's safe to blank the pane only because `showClearContextOnPlanAccept` stays default-false; compaction renders as a `notice`-role divider with plain (non-`toLocaleString`) token counts for deterministic tests. |
| 2026-07-19 | [Packaging: unpack the SDK's Claude CLI *and* point at it](reasoning/2026-07-19-packaging-asarunpack-claude-cli.md) | Getting a packaged pane's session to start took three fixes, not one: (1) externalize the SDK out of the bundled main via `ssr.external` (Vite SSR bundled it despite rollup `external`; keep `external: ['electron']` too or the ESM main crashes on `__dirname`); (2) `asarUnpack` the native binary package; (3) pass `pathToClaudeCodeExecutable` rewriting `app.asar`→`app.asar.unpacked`, because the SDK's `import.meta.url` resolver otherwise points at the asar *virtual* path ("exists but failed to launch"). | 2026-07-19 | [Standard `scrollbar-width` disables `::-webkit-scrollbar`](reasoning/2026-07-19-scrollbar-width-disables-webkit-styling.md) | Chromium ignores every `::-webkit-scrollbar` rule on an element that also carries the standard `scrollbar-width`/`scrollbar-color`. `shadcn/tailwind.css` ships `scrollbar-thin` (→ `scrollbar-width: thin`) that collides with a same-named custom `@utility`, silently winning and reverting the bar to native. To draw a rounded inset pill on a native-scroll element, use a distinctly-named webkit-only utility (`scrollbar-app`) carrying no standard `scrollbar-width`; grep the compiled CSS for `scrollbar-width` when styling "does nothing." |
| 2026-07-19 | [File-checkpoint rewind: one uuid, two mechanisms](reasoning/2026-07-19-file-checkpoint-rewind-uuid-composition.md) | One user-turn uuid drives both `rewindFiles` (files) and `resumeSessionAt`+`forkSession` (conversation); `replay-user-messages` extraArg is mandatory to receive it; rewind files on the live query *before* forking; live replay-stream uuid and restored `SessionMessage.uuid` are assumed one namespace (restart-survival); correlate checkpoints FIFO.
